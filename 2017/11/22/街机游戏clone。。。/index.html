  • clientX,clientY,screenX,screenY,offsetX,offsetY
    他们是鼠标事件对象的几个属性，只有鼠标事件发生时才会有意义。

    clientX：当鼠标事件发生时（以下都有这句），鼠标相对于浏览器（浏览器的有效区域）x轴的位置；clientY就是y轴的位置。
    screenX：鼠标相对于显示器屏幕x轴的位置；screenY是显示屏屏幕y轴的位置。y值通常与client不同
    offsetX：鼠标相对于事件源x轴的位置；offsetY是y轴的位置。某个元素上相对于该元素0，0的距离。

插播：<span style = "white-space:pre"></span>这个css的white-space属性设置如何处理元素内的空白。pre表示空白会被浏览器保留，行为方式像HTML中的<pre>标签。

  • 处理键盘输入的插件：https://github.com/marquete/kibo
    键码对照表：https://segmentfault.com/a/1190000005828048
  • requestAnimationFrame
    不需设置时间间隔，与浏览器的刷新频率相同，不会导致过度绘制出现的第三帧丢失。而且它采用系统时间间隔，更准确且流畅。
