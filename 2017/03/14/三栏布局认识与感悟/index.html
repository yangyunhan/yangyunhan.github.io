三栏布局认识与感悟

发表于 2017-03-14 | 分类于 skill |
两边栏宽度固定，中间栏宽度自适应，高度为最高的一栏。

为了完成百度前端学院的第三个任务，这几天找了很多关于布局和高度一致的文章和材料。到现在为止，刚刚把问题解决掉。发现自己在做这个任务的时候，大方向上的顺序错了。应该是先把框架搭建起来，再添加内部的细节样式。当然，也是因为我对布局的了解还不够全面和清晰。光做到左右宽度固定，中间自适应的框架和同时做到高度自适应还是不一样的。下面我对这个任务所涉及到的布局简单总结一下。
如果是只实现两边固定，中间自适应，那么下面这个博客就介绍的很清楚。
两边固定，中间自适应布局
如果在这个基础上再做到高度自适应就会很难，应该这些方法从左到右依次安排子元素，并且都在子元素上使用position:absolute或者float:left|right。这样使得子元素脱离了父元素，造成有的元素没有被包含在父元素中，只是起到了定位的作用。在师兄的帮助下，我了解到了圣杯布局、双飞翼布局，可以很好的解决这个问题。而flex是一个更简介的方式帮助实现这些布局。
我是根据一篇博客的介绍，把代码粘过来对比发现两种方法的不同的。
圣杯布局与双飞翼布局
文章中说两种方法都可以使得高度达到自适应的效果，但事实上，我在火狐浏览器上并没有看到自适应的效果，这是一个遗留问题。可能会与浏览器的兼容性有关，但我现在不打算讨论这个问题。
这两个布局的共同特点就是先把中间重要的内容列出来，先出现在DOM树上，然后再把两个元素通过margin负值分别定位到应该出现的位置上。不同点是我认为圣杯布局更符合构建一个DOM树的习惯，由一个大容器包裹很多个小容器，每个小容器并列，用positive:relative和left\top去定位兄弟元素位置；而双飞翼布局三个元素外面没有父元素，直接出现在body元素下，中间的部分外面嵌套一层div，这个方法不用positive，只用float和margin负值法定位。所以在这里，圣杯布局更能满足我的需求，因为通过给三个元素共有的父元素设置背景颜色，子元素不设置颜色达到效果。
